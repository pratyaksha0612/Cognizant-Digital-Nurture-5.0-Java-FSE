# Week 07 - Version Control with Git

## Overview

This week focuses on the fundamentals of Git version control, including repository configuration, ignoring unnecessary files, branching, merging, conflict handling, and synchronizing local repositories with remote repositories.

## Exercises

### Exercise 01 - Git Configuration

Configured Git for local development by:

- Installing and verifying Git
- Configuring user information
- Setting Visual Studio Code as the default editor
- Initializing a Git repository
- Creating and committing files

### Exercise 02 - Git Ignore

Implemented Git ignore functionality by:

- Creating a `.gitignore` file
- Ignoring log files
- Ignoring directories
- Verifying ignored files using Git status

### Exercise 03 - Branching and Merging

Performed branch management operations including:

- Creating feature branches
- Switching between branches
- Committing changes
- Comparing branches
- Merging branches
- Viewing Git history

### Exercise 04 - Merge Conflict Resolution

Explored Git merge operations by:

- Creating independent branches
- Performing merge operations
- Understanding Fast-forward merges
- Learning when merge conflicts occur

### Exercise 05 - Push to Remote Repository

Demonstrated synchronization between local and remote repositories by:

- Checking repository status
- Staging files
- Committing changes
- Pushing commits to GitHub

## Technologies Used

- Git
- Git Bash
- GitHub
- Visual Studio Code

## Learning Outcomes

After completing this week, I gained practical experience in:

- Configuring Git for development
- Managing local repositories
- Tracking files using Git
- Ignoring unnecessary files with `.gitignore`
- Working with branches
- Merging code changes
- Understanding merge conflicts and Fast-forward merges
- Managing commit history
- Synchronizing local repositories with GitHub
